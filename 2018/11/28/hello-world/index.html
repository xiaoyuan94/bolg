Hello World ！新的博客

发表于 2018-11-28 | 分类于 开发

简述

再次迁移一下博客，用的阿里的主机用来学习和部署一些东西，快到期了，一晃也用了一年半了，已经有人打电话让我再续费了，于是决定放弃了这个模式。

没有太多时间来维护，另外一个最主要的问题是花钱啊！！！都没什么时间玩了，还花个毛钱，况且阿里的还老是涨价，指望便宜的主机也不是总是有。

==反正是花钱是不可能再花钱的。==

之前的方案：主机+域名 RMB:（搞活动的主机，现价880）360+60

现在的方案：0元或者只需要个域名钱

过几天服务器到期了，让域名解析到这个上面就彻底OK了。

网站的架构;

hexo + next 主题，其他的配置参考next配置

githup的账户，开通一个仓库用来作为page界面，配置关联hexo静态即可完成

最后将自己的编辑完毕的东西统一推送到githup上面page界面即可

PS:当然可以使用国内的page方案了，githup有时候还是有点慢的，例如gitee也是可以的，但我个人感觉githup更好，毕竟更专业点。

优缺点

优点:

不花钱已经是最大的优点了，githup不倒闭你的这个就可以永远用下去，还有人替你维护服务器，免去了自己维护的成本。

缺点:

1，流量有限制，个人博客也不需要多少流量，够用了。

2，没有数据库，只能是静态界面，扩展比较单一，不适合任何一个企业级web网站。

3，静态界面的博客网站的构建方案对比WordPress之类的还是有所差距，但个人感觉已经够用了。

搭建流程

网上很多教程了，但是感觉不怎么统一，很多只说了一半，但是都比较清晰，这里只介绍步骤，其中细节百度一下吧，它们肯定比我这个更加详细。

1，首先肯定要Hexo搭建一下，使用npm来安装，几步就ok了。

https://hexo.io/zh-cn/docs/

2，Hexo安装完毕以后，使用hexo s 来本地运行一下，监听的是4000端口，本地看下效果。

3，本地运行没有问题以后，然后要转到githup上面去配置了，建个仓库配置一下主题，让这个仓库作为io结尾的的page界面，然后为这个仓库配置ssh，便于hexo的推送界面给这个仓库，测试一下githup提供的域名

4，上面的都ok以后先不要着急hexo -d -s 推送到仓库当中，我们需要选择个好看好用一点的主题模板是不是，我使用的是next主题

http://theme-next.iissnan.com/

5, 主题下载完毕以后，按照它上面的配置一遍，很多很多功能，基本一个博客需要的应该都有了，很不错。

6，最后，注意，注意，很多失败都是亡于最后一步，这个时候已经要记得修改主配置里面的域名和root，不然本地查看没有问题，但是你一推送上去就完蛋。

完毕，收工！
